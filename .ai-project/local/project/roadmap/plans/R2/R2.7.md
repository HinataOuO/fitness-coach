## Implementation instructions
- Prima di eliminare ogni file `phases/*.md`, confrontare voce per voce contenuto destinato a restare con skill R2 competente. Se manca una regola, trasferirla senza cambiarne significato; eliminare fase solo dopo confronto superato.
- Aggiornare README con sintassi `$fitness-coach <skill> <Nome>`, quattro skill interne, struttura `Profiles/<Nome>/`, history per-file e pipeline JSON→Python→HTML.
- Aggiornare issue template da fasi/file profilo legacy a skill e percorsi R2.
- Rimuovere riferimenti operativi a directory atleta top-level, `profile-core.md`, `profile-plan-current.md`, `profile-log-history.md`, `profile-plans-archive.md`, `phases/*`, `generate-week-plan` e `artifacts/<slug-atleta>`.
- Eliminare senza destinazione: reminder autonomo domenicale; follow-up dopo 7/14 giorni o 2 settimane; revisioni proattive ogni 2–4 settimane; re-profilazione; offerta automatica HTML dopo profiling.
- La rimozione delle revisioni `2–4 settimane` non elimina test tecnici di mobilità o benchmark quando parte esplicita del piano; elimina solo revisione autonoma del piano madre prima della scadenza ciclo.
- Conservare integralmente: export atleta selezionato; sicurezza e gestione infortuni; confine nutrizione/`meal-planner`; ricerca scientifica e citazioni; regole tecniche di profiling, planning, analisi e generazione non escluse sopra.
- Non aggiungere fallback, alias o documentazione duale legacy/R2.

## Verification
- Eseguire confronto checklist R2.0 prima/dopo e registrare ogni regola conservata o eliminata intenzionalmente.
- Eseguire `rg -n 'phases/|profile-core|profile-plan-current|profile-log-history|profile-plans-archive|generate-week-plan|artifacts/<slug-atleta>|backupPietro' --glob '!\.ai-project/local/project/roadmap/**' --glob '!\.git/**' .` e richiedere zero riferimenti operativi legacy.
- Eseguire ricerche mirate per `7 giorni`, `14 giorni`, `2 settimane`, `revisioni.*2.*4`, `re-profil` e offerta HTML automatica; accettare solo regole tecniche non vietate esplicitamente.
- Verificare manualmente README e issue template contro struttura R2.

## Done criteria
- File fase sono rimossi solo dopo trasferimento verificato.
- Documentazione e issue template descrivono esclusivamente R2.
- Comportamenti autonomi vietati sono assenti.
- Export, sicurezza, nutrizione e ricerca scientifica sono preservati.
- Nessun riferimento operativo a percorsi legacy resta fuori dallo storico roadmap.
- Nessun file fuori da `Allowed files` modificato.
