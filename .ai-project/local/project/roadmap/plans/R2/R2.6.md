# R2.6 Estrarre skill plan

## Title
Estrarre skill interna `plan`.

## Goal
Generare artefatti settimanali atleta tramite JSON validato e generatore Python, eliminando comando pubblico legacy.

## Layer
backend

## Skill
domain-feature

## Allowed files
- `.agents/skills/fitness-coach-plan/SKILL.md`
- `.agents/skills/generate-week-plan/SKILL.md` (rimozione)
- `.claude/commands/generate-week-plan.md` (rimozione)

## Out of scope
- Modifiche a schema JSON, template HTML o generatore Python.
- Modifiche a `profile.md`, `plan.md` o history.
- Scrittura manuale di HTML.
- Nuovo wrapper pubblico per `plan`.

## Context to load
- `.ai-project/runtime/skills/domain-feature/SKILL.md`
- `.ai-project/runtime/skills/ponytail/SKILL.md`

- `references/profile-schema.md`
- R2.1–R2.5 completati.

## Implementation instructions
- Creare skill interna conforme alla forma skill progetto e trasferire contratto utile di `generate-week-plan`.
- Richiedere atleta selezionato; leggere `profile.md`, `plan.md` e `history/last-week.md` solo quanto necessario.
- Derivare settimana esclusivamente da `Plan week` in `profile.md`; non accettare un numero settimana concorrente inventato o separato.
- Applicare piano madre più adattamenti dell'ultima analisi; W1 può essere generata senza `last-week.md`, W2+ lo richiede.
- Scrivere temporaneo JSON conforme a `assets/week-plan.schema.json` dentro `Profiles/<Nome>/artifacts/`; validarlo con `python3 scripts/generate_week_plan.py <temp-json>`.
- Quando richiesto contratto HTML, eseguire esclusivamente `python3 scripts/generate_week_plan.py <temp-json> --output <temp-html>`. Agente non deve scrivere o modificare HTML.
- Pubblicare JSON/HTML nella directory artifacts solo dopo uscita zero, con sostituzioni atomiche. Su qualsiasi errore rimuovere temporanei e lasciare artefatti precedenti invariati.
- Mantenere JSON come fonte canonica e garantire che HTML pubblicato derivi dallo stesso JSON.
- Rimuovere skill pubblica `.agents/skills/generate-week-plan/` e wrapper `.claude/commands/generate-week-plan.md`; nessun alias permanente.

## Verification
- Validare nuova skill e verificare assenza wrapper/skill pubblica legacy.
- Simulare W1, W2+, `Plan week` mancante, JSON invalido e fallimento generatore.
- Verificare che comando Python sia unico percorso HTML e che output resti sotto `Profiles/<Nome>/artifacts/`.
- Confrontare artefatti preesistenti dopo errori; devono restare byte-identici e senza temporanei residui.

## Done criteria
- `plan` usa solo `Plan week` del profilo.
- JSON viene validato prima della pubblicazione.
- HTML viene prodotto solo dallo script Python dallo stesso JSON.
- Errori preservano artefatti precedenti.
- Skill e wrapper `generate-week-plan` non esistono più.
- Nessun file fuori da `Allowed files` modificato.
