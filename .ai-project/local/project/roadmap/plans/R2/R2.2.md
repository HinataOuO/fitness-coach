# R2.2 Rifattorizzare dispatcher DONE

## Title
Rifattorizzare dispatcher fitness.

## Goal
Trasformare `SKILL.md` in router minimo verso quattro skill interne, selezionando atleta prima di leggere dati persistiti.

## Layer
backend

## Skill
domain-feature

## Allowed files
- `SKILL.md`

## Out of scope
- Implementazione dei protocolli `profile`, `planning`, `analyze` e `plan`.
- Modifiche a profili, schema, fasi, riferimenti, generatore o test.
- Compatibilità permanente con sintassi o percorsi legacy.

## Context to load
- `.ai-project/runtime/skills/domain-feature/SKILL.md`
- `.ai-project/runtime/skills/ponytail/SKILL.md`

- R2.1 completato.

## Implementation instructions
- Ridurre `SKILL.md` a routing, selezione atleta, export e limiti condivisi; rimuovere logica operativa duplicata.
- Definire sintassi canonica `$fitness-coach <skill> <Nome>` con `<skill>` in `profile`, `planning`, `analyze`, `plan`.
- Mappare ogni comando alla corrispondente `.agents/skills/fitness-coach-<skill>/SKILL.md`.
- Se `<skill>` manca o non è valido, mostrare solo sintassi e quattro skill disponibili.
- Se `<Nome>` manca, elencare esclusivamente nomi delle directory immediate sotto `Profiles/`, senza aprire alcun file atleta; poi chiedere selezione.
- Risolvere `<Nome>` a una singola directory immediata `Profiles/<Nome>/`; vietare traversal, ricerca fuzzy e inferenza leggendo profili.
- Non leggere `profile.md`, `plan.md`, history o artifacts prima che skill e atleta siano selezionati.
- Non rispondere a richieste fitness generiche senza profilo selezionato: richiedere comando e atleta. Per nuovo atleta, instradare a `profile` con nome esplicito.
- Conservare integralmente export del solo atleta selezionato e blocchi di sicurezza: niente diagnosi/farmaci/steroidi, nutrizione dettagliata a `meal-planner`, ricerca con fonti primarie autorevoli, comunicazione diretta e non giudicante.

## Verification
- Simulare comando senza skill, senza atleta, atleta inesistente, più profili e comando valido per ciascuna skill.
- Confermare tramite ispezione che listing legge solo nomi directory e nessun dato atleta.
- Verificare rifiuto di `../`, nomi ambigui e domanda fitness generica senza selezione.
- Verificare export limitato a `Profiles/<Nome>/profile.md` e `plan.md`.

## Done criteria
- Dispatcher contiene routing, non protocolli tecnici.
- Sintassi `$fitness-coach <skill> <Nome>` è unica e documentata.
- Nessun dato atleta viene letto prima della selezione.
- Export e limiti di sicurezza restano presenti.
- Nessun file fuori da `Allowed files` modificato.
