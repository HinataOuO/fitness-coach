# R2.5 Estrarre skill analyze

## Title
Estrarre skill interna `analyze`.

## Goal
Analizzare report del diario, mantenere ultimo report e history settimanale, aggiornando profilo senza alterare piano madre.

## Layer
backend

## Skill
domain-feature

## Allowed files
- `.agents/skills/fitness-coach-analyze/SKILL.md`

## Out of scope
- Modifiche o rimozione di `phases/monitoring.md`.
- Revisione del piano madre.
- Generazione della scheda settimanale JSON/HTML.
- Reminder autonomi, follow-up temporizzati o re-profilazione.

## Context to load
- `.ai-project/runtime/skills/domain-feature/SKILL.md`
- `.ai-project/runtime/skills/ponytail/SKILL.md`
- `.ai-project/local/project/PROJECT_INDEX.md`

- `references/profile-schema.md`
- `references/recovery-and-deload.md`, `references/common-injuries.md`, `references/mobility-and-flexibility.md` o altri riferimenti, solo se dati report li rendono necessari.
- R2.1–R2.4 completati.

## Implementation instructions
- Creare skill conforme alla forma skill progetto.
- Accettare report settimanale completo prodotto dal diario; se incompleto o settimana incoerente, chiedere dati mancanti prima di scrivere.
- Leggere solo `Profiles/<Nome>/profile.md`, `plan.md`, `history/last-week.md` se presente e riferimenti strettamente necessari. Non caricare history completa.
- Conservare regole tecniche pertinenti di monitoraggio: interpretazione volume/RPE/dolore/fatica/aderenza, plateau, sicurezza infortuni, mobilità, comunicazione e fonti scientifiche su richiesta.
- Produrre risposta di analisi e salvare report completo, non un riassunto lossy, in Markdown compatto con settimana, date, dati giornalieri, metriche, dolore/mobilità, note e analisi coach.
- W1: scrivere atomicamente nuovo `history/last-week.md`; nessun archivio precedente richiesto.
- W2+: prima archiviare `history/last-week.md` in `history/weeks/W<N>-<YYYY-MM-DD>.md` senza sovrascrivere; poi pubblicare nuovo `last-week.md`. Se qualsiasi passo fallisce, preservare stato precedente.
- Aggiornare in `profile.md` solo valori correnti confermati: peso, benchmark, `Last log`, `Plan week` e flags necessari. Non riscrivere dati storici nel profilo.
- Non modificare `plan.md`. Adattamenti consigliati riguardano prossima settimana e saranno consumati da `plan` tramite ultimo report.

## Verification
- Simulare report incompleto, W1, W2+ e nome archivio già esistente.
- Verificare letture limitate a profilo, piano, ultima settimana e riferimenti necessari.
- Verificare round-trip del report: nessun dato del diario perso nel Markdown salvato.
- Confrontare `plan.md` byte-per-byte prima/dopo analisi.
- Validare frontmatter e sezioni skill.

## Done criteria
- Report completo è salvato in formato compatto.
- W1 e rotazione W2+ funzionano senza perdita o sovrascrittura.
- Profilo e `Plan week` sono aggiornati con dati confermati.
- Piano madre resta invariato.
- Nessun file fuori da `Allowed files` modificato.
