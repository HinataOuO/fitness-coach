# R1.3 Creare generatore Python

## Title
Creare generatore Python per piano settimanale.

## Goal
Generare atomicamente un HTML standalone da un piano JSON valido usando sola standard library Python.

## Layer
backend

## Skill
backend

## Allowed files
- `scripts/generate_week_plan.py`
- `assets/week-plan.schema.json`
- `.gitignore`

## Out of scope
- Modifiche grafiche o comportamentali a `assets/fitness-coach-log.html`.
- Logica coaching e creazione del contenuto del piano.
- Comando agente `generate-week-plan` e wrapper Claude.
- Creazione o commit di file sotto `artifacts/`.

## Context to load
- `.ai-project/runtime/skills/backend/SKILL.md`
- `.ai-project/runtime/skills/ponytail/SKILL.md`
- `.ai-project/local/project/PROJECT_INDEX.md`
- `assets/week-plan.schema.json`
- `assets/fitness-coach-log.html`

## Implementation instructions
- Creare `scripts/generate_week_plan.py` con interfaccia `python3 scripts/generate_week_plan.py PLAN.json --output PLAN.html`.
- Usare sola standard library; leggere template da `assets/fitness-coach-log.html` e contratto da `assets/week-plan.schema.json`.
- Validare prima della generazione campi obbligatori, tipi, versione del contratto, strutture annidate e varianti esercizio. Se lo schema usa costrutti non supportati direttamente dalla standard library, implementare controlli espliciti coerenti con esso.
- Richiedere esattamente un marker nel template; fallire chiaramente se assente o duplicato.
- Serializzare e iniettare payload impedendo che qualsiasi valore possa chiudere anticipatamente il tag `<script>`; preservare equivalenza dei dati dopo parsing nel browser.
- Scrivere file temporaneo nella directory destinazione e sostituire output solo dopo completamento riuscito. Su errore, non lasciare HTML parziale e non corrompere output preesistente.
- Aggiungere a `.gitignore` commento `# Generated plans may contain personal training data` e regola `/artifacts/`.

## Verification
- Eseguire CLI con JSON valido e verificare output standalone.
- Verificare rifiuto di input incompleto, tipi errati, versione non supportata, template senza marker e template con marker duplicato.
- Verificare escaping di un valore contenente `</script>` e assenza del marker nell'HTML prodotto.
- Simulare errore di generazione e verificare che output preesistente resti intatto e nessun temporaneo residuo rimanga.
- Eseguire controllo sintattico Python con `python3 -m py_compile scripts/generate_week_plan.py`.

## Done criteria
- CLI pubblica funziona con firma documentata e sola standard library.
- Input invalido fallisce prima di produrre output.
- Iniezione è sicura rispetto a chiusura anticipata di `<script>`.
- Scrittura è atomica.
- `/artifacts/` è ignorata con motivazione privacy.
- Nessun file fuori da `Allowed files` modificato.
