# R1.5 Verificare flusso completo

## Title
Verificare flusso completo di generazione settimanale.

## Goal
Dimostrare che comando, contratto JSON, generatore e diario HTML funzionano insieme nei casi validi e falliscono in sicurezza nei casi invalidi.

## Layer
verify

## Skill
backend

## Allowed files
- `tests/test_generate_week_plan.py`
- `tests/fixtures/week-plan/**`
- `scripts/generate_week_plan.py`
- `assets/fitness-coach-log.html`
- `assets/week-plan.schema.json`
- `.agents/skills/generate-week-plan/SKILL.md`
- `.claude/commands/generate-week-plan.md`
- `phases/planning.md`
- `.gitignore`

## Out of scope
- Nuove funzionalità di coaching.
- Restyling del diario.
- Modifiche del contratto pubblico oltre correzioni necessarie a soddisfare requisiti già pianificati.
- Commit di profili reali o output sotto `artifacts/`.

## Context to load
- `.ai-project/runtime/skills/backend/SKILL.md`
- `.ai-project/runtime/skills/ponytail/SKILL.md`
- `.ai-project/local/project/PROJECT_INDEX.md`
- Task `R1.2`, `R1.3` e `R1.4` completati.
- File elencati in `Allowed files` pertinenti al caso verificato.

## Implementation instructions
- Creare test con standard library Python per JSON valido e input incompleti o tipologicamente errati.
- Coprire almeno un esercizio per ciascuna variante `reps`, `time`, `dist` e `mobility`.
- Testare template senza marker, marker duplicato, payload contenente `</script>`, assenza marker residuo e conservazione esatta del payload dopo parsing.
- Testare atomicità: errore non crea output parziale, non sostituisce output preesistente e non lascia file temporanei.
- Eseguire CLI su fixture completa e verificare JSON sorgente più HTML standalone nei percorsi attesi.
- Fare smoke test browser del risultato: rendering sessioni ed esercizi, uso offline, persistenza dopo refresh e generazione report.
- Verificare con Git che `/artifacts/` sia ignorata, mentre roadmap, schema, script, template e test non lo siano.
- Applicare solo correzioni minime entro `Allowed files` se verifica espone difetti dei task precedenti.

## Verification
- Eseguire `python3 -m unittest tests/test_generate_week_plan.py`.
- Eseguire generatore su fixture completa e ispezionare HTML prodotto per marker residui e payload sicuro.
- Eseguire smoke test in browser senza rete, aggiornare un valore, ricaricare pagina e generare report.
- Eseguire `git check-ignore -v artifacts/example/week-W1.json artifacts/example/week-W1.html` e verificare esito positivo.
- Eseguire `git check-ignore` su roadmap, schema, script, template e test e verificare che non risultino ignorati.

## Done criteria
- Suite copre percorso valido, input incompleti, quattro tipi esercizio, marker ed escaping, scrittura atomica e regole ignore.
- HTML generato funziona offline, persiste dati e genera report nel browser.
- Nessun marker resta nell'output finale.
- Artefatti personali sono ignorati; sorgenti e roadmap restano tracciabili.
- Nessun file fuori da `Allowed files` modificato.
