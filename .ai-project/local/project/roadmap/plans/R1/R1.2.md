# R1.2 Preparare template HTML iniettabile

## Title
Preparare template HTML iniettabile.

## Goal
Rendere il diario HTML inizializzabile da un piano settimanale JSON versionato, mantenendo invariato uso offline, persistenza locale e generazione report.

## Layer
frontend

## Skill
frontend

## Allowed files
- `assets/fitness-coach-log.html`
- `assets/week-plan.schema.json`

## Out of scope
- Generatore Python e relativa CLI.
- Comando agente `generate-week-plan` e wrapper Claude.
- Logica coaching e analisi dei progressi.
- Artefatti JSON o HTML generati.

## Context to load
- `.ai-project/runtime/skills/frontend/SKILL.md`
- `.ai-project/runtime/skills/ponytail/SKILL.md`
- `.ai-project/local/project/PROJECT_INDEX.md`
- `assets/fitness-coach-log.html`

## Implementation instructions
- Definire in `assets/week-plan.schema.json` un contratto JSON con versione esplicita e strutture validate per piano, atleta, settimana, obiettivo, sessioni ed esercizi.
- Modellare nel contratto le varianti esercizio necessarie al diario: `reps`, `time`, `dist` e `mobility`.
- Inserire nel template un unico marker di payload dentro `<script type="application/json">`; il marker deve essere non ambiguo e sostituibile una sola volta dal generatore.
- Sostituire i dati hardcoded di `initDefaults()` con parsing e inizializzazione dal payload incorporato.
- Conservare comportamento standalone/offline, local storage esistente e generazione report.
- Mostrare errore comprensibile se payload incorporato manca o non è interpretabile, senza introdurre dipendenze esterne.

## Verification
- Aprire il template con un payload di esempio valido e verificare rendering di atleta, settimana, obiettivo, sessioni ed esercizi.
- Verificare che refresh mantenga dati tramite local storage e che generazione report continui a funzionare.
- Verificare presenza di un solo marker e assenza dei valori piano hardcoded in `initDefaults()`.
- Validare sintassi di `assets/week-plan.schema.json` con Python standard library.

## Done criteria
- Schema JSON versionato descrive intero payload richiesto e quattro tipi esercizio.
- Template inizializza diario esclusivamente dal payload JSON incorporato.
- Funzioni offline, persistenza e report restano operative.
- Nessun file fuori da `Allowed files` modificato.
