# R1.4 Aggiungere comando generate-week-plan

## Title
Aggiungere comando `generate-week-plan`.

## Goal
Fornire percorso canonico che usa dati atleta e regole fitness esistenti per produrre JSON validato e HTML settimanale standalone.

## Layer
backend

## Skill
domain-feature

## Allowed files
- `.agents/skills/generate-week-plan/SKILL.md`
- `.claude/commands/generate-week-plan.md`
- `phases/planning.md`

## Out of scope
- Modifiche a schema JSON.
- Modifiche a generatore Python o sua validazione.
- Modifiche a HTML, CSS o JavaScript del template.
- Commit di profili atleta o artefatti generati.

## Context to load
- `.ai-project/runtime/skills/domain-feature/SKILL.md`
- `.ai-project/runtime/skills/fitness-coach/SKILL.md`
- `.ai-project/local/project/PROJECT_INDEX.md`
- `.ai-project/local/project/memory/MEMORY_INDEX.md`, poi solo shard fitness pertinenti.
- `phases/planning.md`
- `assets/week-plan.schema.json`
- `scripts/generate_week_plan.py`

## Implementation instructions
- Creare skill canonica `.agents/skills/generate-week-plan/SKILL.md` usando forma skill prevista dalle convenzioni progetto.
- Creare `.claude/commands/generate-week-plan.md` come wrapper sottile che inoltra alla skill canonica, senza duplicarne logica.
- Definire flusso: caricare profilo atleta, piano corrente e ultimo report; analizzare progressi con regole fitness già presenti; produrre JSON conforme allo schema; salvare `artifacts/<atleta>/week-W<N>.json`; invocare CLI Python; produrre `artifacts/<atleta>/week-W<N>.html`.
- Richiedere all'utente dati obbligatori mancanti prima di generare. Non inventare profilo, feedback, settimana o vincoli necessari.
- Sanitizzare segmento `<atleta>` per ottenere percorso locale stabile e impedire traversal; usare numero settimana coerente con dati del piano.
- Trattare JSON come sorgente intermedia autorevole e interrompere flusso se validazione o generazione fallisce.
- Aggiornare `phases/planning.md` indicando `generate-week-plan` come percorso canonico di generazione della scheda settimanale.

## Verification
- Controllare frontmatter e sezioni della skill contro convenzioni progetto.
- Verificare che wrapper Claude punti alla skill canonica senza logica duplicata.
- Simulare caso completo e confermare sequenza JSON prima, generatore poi, HTML finale.
- Simulare dati mancanti e confermare richiesta all'utente prima di qualsiasi artefatto.
- Verificare che percorsi output restino sotto `artifacts/<atleta>/` anche con nome atleta non sicuro.

## Done criteria
- Comando `generate-week-plan` è disponibile tramite skill canonica e wrapper Claude.
- Flusso carica contesto minimo corretto, applica regole fitness esistenti e produce entrambi gli artefatti.
- Dati mancanti o errori di validazione bloccano generazione senza output fuorviante.
- `phases/planning.md` presenta nuovo comando come percorso canonico.
- Nessun file fuori da `Allowed files` modificato.
