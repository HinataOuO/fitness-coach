Portable starter for Codex and Claude project agents.

AI Project Starter installs shared rules, task skills, slash commands, memory templates, roadmap templates, and issue templates. It keeps the update-managed package separate from project-local state.

Overview

This repository is the template source. After installation, the package lives in .ai-project/runtime/. Project-local state lives in .ai-project/local/ and is not overwritten by updates.

Runtime

Core rules, skills, command wrappers, scripts, and templates managed by the package.

Local state

Memory shards, roadmap, overlays, and project-specific facts committed with the project.

Discovery copy

Synced entrypoints: AGENTS.md, CLAUDE.md, .agents/skills/, .claude/commands/.

starter.json defines .ai-project as the canonical root, .ai-project/runtime as update-managed package root, .ai-project/local as project-local root, plus 16 skills, 16 Claude command wrappers, memory tags, and roadmap layers.

Installation

Fast install into the current repository

curl -fsSL https://raw.githubusercontent.com/HinataOuO/AI-ProjectManager/main/scripts/ai-project.mjs | node - install-here

Install from a local clone or Git source

node /path/to/AI-ProjectStarter/scripts/ai-project.mjs install <git-url-or-local-path> --version v2.0.0

Commit after installation

git add AGENTS.md CLAUDE.md .agents .claude .ai-project.lock.json .ai-project/local
git commit -m "chore: install AI project manager"
Important: commit .ai-project/local/ because it contains project memory, overlays, and roadmap state. Do not hand-edit .ai-project/runtime/; it is managed by the package.

Update installed project and agents

update downloads/replaces .ai-project/runtime/ from the configured source and updates .ai-project.lock.json. It never overwrites .ai-project/local/. 

node .ai-project/runtime/scripts/ai-project.mjs update --version main
node .ai-project/runtime/scripts/ai-project.mjs sync-discovery --force

sync-discovery does not download updates. It regenerates AGENTS.md, CLAUDE.md, .agents/skills/, and .claude/commands/ from the current local runtime. If nothing changes, run update --version main first or check the source/ref with status. 

node .ai-project/runtime/scripts/ai-project.mjs status

Internal Logic

Installation flow

1. SourceMaterialize Git URL or local path. Git sources are cloned into a temp directory.
2. Manifeststarter.json validates canonical root and lists.
3. RuntimeCopy package files into .ai-project/runtime/.
4. LocalSeed project/ and example overlay only if missing.
5. DiscoverySync root entrypoints and skill/command discovery copies.
6. LockWrite source, ref, commit, package version, and runtime hashes.
7. CommitUser commits entrypoints, lock, and local state.
8. UpdateFuture updates overwrite runtime after drift check.

Agent task flow

  1. Load .ai-project/runtime/core/KERNEL.md.
  2. Load .ai-project/runtime/core/CONVENTIONS.md.
  3. Load .ai-project/local/project/PROJECT_INDEX.md.
  4. Select the smallest matching skill or command wrapper.
  5. Read MEMORY_INDEX.md, then only matching tagged shards.
  6. For roadmap work, read index, one MACRO.md, then one layer file.
  7. Read overlay only when project paths, commands, DB provider, or domain rules are needed.
  8. Work narrowly, verify, then propose memory deltas only when durable and explicitly confirmed.

Claude Commands

Files in commands/claude/ are thin wrappers. Each wrapper loads .ai-project/runtime/skills/<name>/SKILL.md and follows its load/scope/deny/procedure/done. 

# /frontend
Load `.ai-project/runtime/skills/frontend/SKILL.md`. Follow its `load/scope/deny/procedure/done`.

Installed copies live in .claude/commands/ and are regenerated by sync-discovery. Project-specific aliases should be created outside the template after installation.

Memory

Local memory lives in .ai-project/local/project/memory/. Agents read MEMORY_INDEX.md first, then only shards with relevant tags.

always backend frontend migration discovery domain roadmap

Required frontmatter

---
id: short-id
tags: [always]
load: always
updated: YYYY-MM-DD
depends: []
---
  • always: user preferences and durable workflow rules.
  • backend: server, DB access, API, auth, validation.
  • frontend: UI, components, styles, i18n messages.
  • migration: schema, migrations, seed/import.
  • discovery: project map, paths, commands, architecture.
  • domain: project domain rules.
  • roadmap: durable planning and completion state.
No automatic memory writes: every memory write requires explicit confirmation. The agent must show target, exact content or precise summary, and reason.

Roadmap

The local roadmap lives in .ai-project/local/project/roadmap/. The model uses feature macros and technical layers.

ItemRole
INDEX.mdCurrent feature list, status, and layer state.
MACRO.mdFeature goal, status, and layer checklist.
database.mdSchema, migration path, seed/import impact, recovery.
backend.mdServer/API entrypoints, validation, auth/RBAC, error behavior.
frontend.mdScreens/components, loading/empty/error states, i18n, responsive behavior.
verify.mdLint, typecheck, tests, manual flow, known gaps.

Status

Supported states: todo, wip, blocked, done. Default rule: one wip feature unless project overlay says otherwise.

Workflow

  • next-task: reads index, first wip or todo feature, macro, then one layer.
  • task-status: read-only overview without layer details.
  • close-task: closes a layer only after explicit confirmation.

Safety

Destructive or irreversible operations, git push, roadmap close, and memory writes require explicit confirmation before writing.
  • task-status and session-status stay read-only.
  • close-task never writes roadmap state without confirmation.
  • push inspects status/diff/branch, blocks .env*, *.key, *.pem, and never force-pushes.
  • Destructive DB operations require confirmation and recovery notes when needed.
  • Project-specific facts belong in overlay or memory, not generic core/skill/command files.
  • Useful edits found in installed runtime should be moved to local state or the source repo, not left as drift.

Maintenance

Content policy

  • Internal prompting, skills, and commands are English.
  • User-facing output defaults to Italian unless explicitly requested otherwise.
  • Files stay small, deduplicated, and lazy-loaded.
  • Skills use fixed headings and no extra sections.
  • No real project names, private aliases, or domain facts in generic files.

Runtime drift

update compares runtime hashes saved in the lock file with current runtime contents. If drift exists, it fails and lists changed files. Use --force only when you want to discard local runtime edits.

Validation

Main validation from the source repository:

bash scripts/validate-starter.sh

Or from a parent directory:

bash AI-ProjectStarter/scripts/validate-starter.sh

What it checks

  • starter.json: roots, string lists, duplicates, required files.
  • Skills: frontmatter, matching name, description, required headings.
  • Claude wrappers: exactly one correct runtime skill target.
  • Memory: frontmatter, valid tags, inline depends list.
  • Adapters: references to .ai-project/runtime/ and .ai-project/local/.
  • Hardcode scan: blocks project-specific terms in generic files.
  • Word count: monitors runtime token budget.
